v3.3 — LATEST March 3, 2026

Emoji Picker, GIF Search & Like System

😊 Emoji Picker: Built a full emoji picker with 8 categorized tabs (Faces, Hands, Hearts, Animals, Food, Sports, Transport, Objects). Click the smiley face in the input bar to browse and insert emojis.

🎬 GIF Search: Integrated the Tenor GIF API (Google's GIF service) with a live search panel. Click "GIF" in the input bar, search for anything, and click a GIF to send it inline in the chat. GIFs render as animated images directly in the conversation.

❤️ Like System: Replaced the old upvote/downvote reputation system with a clean like-only system. Hover over any message to see a heart "Like" button. Like counts show as ❤️ N next to the username. Hidden when zero for a clean look.

🤖 Gemini-Assisted Breakdown

// HOW GIF MESSAGES WORK:

1. When you click a GIF, the client sends a chat message formatted as [gif:https://url-to-gif]

2. The renderMessage() function in app.js uses a regex to detect this pattern

3. If matched, it renders an <img> tag instead of a text bubble

4. The GIF search uses a server-side proxy at /api/gifs to keep the API key secret

// HOW THE EMOJI PICKER WORKS:

1. Emojis are stored in a JS object emojiCategories — no API needed, pure client-side

2. Each category key is the tab icon (e.g. 😊, 👋, ❤️)

3. Clicking an emoji inserts it at the cursor position in the textarea

v3.2 March 3, 2026

Dynamic Chat Bubbles & Auto-Login

💬 Chat Bubble Polish: Redesigned message bubbles to dynamically size based on content. Short messages like "Hi!" now show tight, compact bubbles instead of stretching across the screen. Uses CSS width: fit-content with a max-width cap.

🔑 Auto-Login: Added JWT token persistence. When you log in, your session is saved to localStorage. On page refresh, the app decodes the JWT, checks expiration, and auto-connects you without showing the login screen.

🤖 Gemini-Assisted Breakdown

// JWT AUTO-LOGIN FLOW:

1. User logs in → server creates a JWT (JSON Web Token) with username, role, expiration

2. Token stored in localStorage.setItem('chat_token', token)

3. On page load → app reads token, decodes the Base64 payload

4. Checks payload.exp * 1000 > Date.now() → if valid, skips login screen

5. If expired or malformed → clears token, shows login modal

// FIT-CONTENT TRICK:

CSS width: fit-content makes a block element shrink to its content width

Combined with max-width: min(80%, 520px) on the parent to cap growth

v3.1 March 2, 2026

Fly.io Migration & munn.fun Domain

☁️ New Hosting: Migrated from Render's free tier to Fly.io with a persistent 1GB SSD volume for the SQLite database. This means user accounts and chat history survive server restarts and redeployments.

🌐 Custom Domain: Purchased munn.fun via Namecheap and configured DNS A records pointing to our Fly.io dedicated IPv4 address. SSL certificates auto-provisioned via Let's Encrypt.

🐳 Dockerized: Added a Dockerfile so the application builds and deploys as a container image. Fly.io builds the Docker image remotely and deploys it to their edge network.

🤖 Gemini-Assisted Breakdown

// WHY FLY.IO OVER HEROKU/RENDER:

Heroku and Render use ephemeral filesystems — files deleted on every restart

Our app uses SQLite (a file-based database) → needs permanent disk

Fly.io supports persistent volumes — a real SSD mounted at /data

Server reads DB_PATH env var: process.env.DB_PATH || './chat.db'

// DNS EXPLAINED:

DNS = Domain Name System — translates "munn.fun" → IP address "66.241.125.192"

We added an A Record at Namecheap: @ → 66.241.125.192

When you type munn.fun, your browser asks DNS for the IP, then connects to Fly.io

⚡ How munn.fun Works

🖥️

Your Browser

HTML + CSS + JavaScript

Renders the UI, handles clicks, sends/receives messages via WebSocket

⟵ ⟶
⟵ ⟶
⚙️

Node.js Server

Express + Socket.io

Handles authentication, routes messages between users, serves files

🗄️

SQLite Database

Persistent file-based storage

Stores user accounts, passwords (hashed), reputation scores, and settings

🟢

Node.js

JavaScript Runtime

Runs JavaScript on the server (outside of a browser). Created by Ryan Dahl in 2009. Used by Netflix, LinkedIn, PayPal, and NASA. It's the engine that powers our chat server.

⚡

Express.js

Web Framework

A minimal web framework that handles HTTP requests (like when you visit a URL). It decides what HTML, CSS, or JSON data to send back. Think of it as the receptionist answering phone calls.

🔌

Socket.io

Real-Time Engine

Creates a persistent two-way connection (WebSocket) between your browser and the server. Unlike normal HTTP which is "ask → answer → done", WebSockets stay open so messages appear instantly without refreshing.

🗃️

SQLite3

Database

A lightweight database that stores data in a single file (chat.db). Unlike big databases like PostgreSQL or MySQL, SQLite needs zero setup — just a file on disk. Perfect for small-to-medium apps. Used in every iPhone and Android phone.

🔐

JWT + bcrypt

Authentication

bcrypt hashes your password so even if the database gets stolen, nobody can read your password. JWT (JSON Web Token) is a signed "passport" the server gives you after login — your browser shows it on each request to prove identity.

✨

Google Gemini 2.5 Flash

AI Integration

Google's generative AI model that powers the @gemini in-chat assistant and the Snow Day Predictor. The server sends prompts to Gemini's API and streams the response back to the chat as a "bot" message.

🐳

Docker + Fly.io

Deployment

Docker packages the entire app into a "container" — a portable box with all dependencies included. Fly.io runs that container on a server in Secaucus, NJ with a persistent SSD volume for our database file.

🎨

TailwindCSS (CDN)

Styling Framework

Instead of writing custom CSS files, Tailwind lets you style elements using utility classes directly in HTML. class="text-white bg-indigo-600 rounded-xl px-4 py-2" — each class does one small thing, and you compose them together.
